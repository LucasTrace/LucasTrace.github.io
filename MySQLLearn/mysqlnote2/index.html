MySQL note-TWO

1.Common Commands and Statements of MySQL

查看表结构：desc nameoftable;

查看表数据：select * from emp;

查看当前使用的数据库：select database();

查看当前使用的数据库版本：select version();

结束正在编写的语句：\c

退出MySQL：\t or quit or exit

查看创建表的语句：show create table nameoftable

简单查询语句(DQL)：select 字段名1,字段名2,… from nameoftable

​ （字段可作简单四则运算，且不区分大小写，但是对应的输出也会改变大小写）

​ 修改sal * 12为 AnnualSal :

查询全部字段：

条件查询：select 字段1,字段2… from nameoftable where condition

执行顺序：1.from 2.where 3.select

in 等同于 or：

模糊查找：select 字段1 from nameoftable where dimcondition

%表示任意多个字符， _表示任意一个字符

排序（升序、降序）select 字段1 from nameoftable order by 字段2;

默认是升序

asc升序-ascend desc降序-descend

执行顺序：1.from 2.where 3.select 4.order

单行处理函数

特点：在数学表达式中只要有null参加运算，结果一定是null

ifnull函数可以对null数据进行预处理

多行处理函数（分组函数）

特点：自动忽略null，不需要额外添加过滤条件

SQL语句中，分组函数不可以出现在where语句中，原因：分组函数在group by之后使用，group by在where后使用

count计数

sum求和

avg平均值

max最大值

min最小值

Group by和having

Group by：按照某个字段或者某些字段进行分组

having：对分组后的数据进行再次过滤

分组函数常和group by联合使用，这也是为什么他被称为分组函数的原因

并且任何一个分组函数（count sum avg max min）都是在group by语句中执行结束之后才会执行的。

当一条SQL语句没有group by的话，整张表的数据都会自成一组

Key Summary：

DQL Statement: select……from……where……group by……having……order by……

执行顺序：1.from 2.where 3.group by 4.having 5.select 6.order by

（可缺少，但是不可改变相对顺序）
