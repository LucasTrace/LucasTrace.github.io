Swagger-note

1.The Definition of Swagger

Swagger 是一个规范和完整的框架，用于生成、描述、调用和可视化 RESTful 风格的 Web 服务。

在项目中使用Swagger需要SpringFox，包括swagger2和ui

2.Springboot集成Swagger前期准备

新建一个springboot web项目后，导入依赖

新建一个config文件夹，新建SwaggerConfig类

至此完成基本配置，进入http://localhost:8080/swagger-ui.html页面即可测试请求。

3.配置Swagger

根据项目的具体情况，配置swagger-ui.html页面中需要显示的内容

4.配置扫描接口以及开关

在application.properties中选定激活的profile名字

分别在application-dev.properties和application-prod.properties中确定端口号

5.接口注释

实体类entity/pojo中可以在类上加@ApiModel(“XXX”)，在属性上加@ApiModelProperty(“XXX”)，便于理解

在controller中，可以在控制方法前加上@ApiOperation(“XXX”)，以说明是何种controller方法

6.swagger小结

  1. 我们可以通过Swagger给一些比较难以理解的属性或者接口增加注释信息，便于理解、调试和后期维护。
  2. 接口文档实时更新，便于调试
  3. 在线测试网页，提高debug效率
  4. 在Web项目正式上线时，需要将Swagger服务关闭，避免接口暴露和内存浪费。
