Regular Expression-note

Regular Expression是一个时常听到且重要的知识点，并且近期对前端的学习过程中，也有用到Regular Expression来检验约束用户的输入字符串，但是一直都没有进行一个系统且全面的Regular Expression学习总结。本篇Blog梳理了Regular Expression的常用知识点，以飨读者，亦供自己日后复习查阅。

The Definition 定义

A regular expression is a pattern that is matched against a subject string from left to right. Regular expressions are used to replace text within a string, validating forms, extracting a substring from a string based on a pattern match, and so much more. The term “regular expression” is a mouthful, so you will usually find the term abbreviated to “regex” or “regexp”.

正则表达式是对字符串操作的一种逻辑公式，就是用事先定义好的一些特定字符、及这些特定字符的组合，组成一个“规则字符串”，这个“规则字符串”用来表达对字符串的一种过滤逻辑。

Basic Matchers 基本匹配

正则表达式其实就是在执行搜索时的格式，它由一些字母和数字组合而成。 例如：一个正则表达式 the，它表示一个规则：由字母t开始，接着是h，再接着是e。
同时正则表达式是大小写的敏感的，所以the不会匹配The.

Meta Characters元字符

正则表达式的核心就是元字符的匹配，元字符的描述如下表所示。

元字符 含义
. Period matches any single character except a line break.句号匹配任意单个字符除了换行符。
[] Character class. Matches any character contained between the square brackets.字符种类。匹配方括号内的任意字符。
[^ ] Negated character class. Matches any character that is not contained between the square brackets否定的字符种类。匹配除了方括号里的任意字符
* Matches 0 or more repetitions of the preceding symbol.匹配>=0个重复的在*号之前的字符。
+ Matches 1 or more repetitions of the preceding symbol.匹配>=1个重复的+号前的字符。
? Makes the preceding symbol optional.标记?之前的字符为可选。
{n,m} Braces. Matches at least “n” but not more than “m” repetitions of the preceding symbol.匹配num个大括号之前的字符或字符集(n <= num <= m)。
(xyz) Character group. Matches the characters xyz in that exact order.字符集，匹配与xyz完全相等的字符串。
| Alternation. Matches either the characters before or the characters after the symbol.或运算符，匹配符号前或后的字符。
\ Escapes the next character. This allows you to match reserved characters [ ] ( ) { } . * + ? ^ $ \ | 转义字符,用于匹配一些保留的字符 [ ] ( ) { } . * + ? ^ $ \
^ Matches the beginning of the input.从开始行开始匹配.
$ Matches the end of the input.从末端开始匹配.

The Full Stop 点运算符

.是元字符中最简单的例子。 .匹配任意单个字符，但不匹配换行符。

“.ar” => The car parked in the garage.

Character Sets 字符集

[ ]方括号用来指定一个字符集。在[ ]方括号中使用连字符来指定字符集的范围。
在方括号中的字符集不关心顺序。例如，表达式[Tt]he 匹配 the 和 The。

“[Tt]he” => The car parked in the garage.

Negated Character Sets 否定字符集

一般来说 ^ 表示一个字符串的开头，但它用在一个方括号的开头[^]的时候，它表示这个字符集是否定的。 例如，表达式[^c]ar 匹配一个ar前面非c的任意字符。如下面的par、gar而非car。

“[ ^c ]ar” => The car parked in the garage.

Repetitions 重复次数

后面跟着元字符 +，* or ? 的，用来指定匹配子模式的次数。 这些元字符在不同的情况下有着不同的意思。

The Star *

*号匹配在 *之前的字符出现大于等于0次。 例如，表达式 a* 匹配0或更多个以a开头的字符。表达式[a-z]* 匹配一个行中所有以小写字母开头的字符串。

“[a-z]*” =>The car parked in the garage.

*字符和.字符搭配可以匹配所有的字符.*。

*和表示匹配空格的符号\s连起来用，如表达式\s*cat\s*匹配0或更多个空格开头和0或更多个空格结尾的cat字符串。

“\scat\s“ => The fat cat on the concatenation.

The Plus +

+号匹配+号之前的字符出现 >=1 次。 例如表达式c.+t匹配以首字母c开头以t结尾，中间跟着至少一个字符的字符串。

“c.+t” => The fat cat sat on the mat.

The Question Mark ?

在正则表达式中元字符 ?标记在符号前面的字符为可选，即出现 0 或 1 次。 例如，表达式 [T]?he 匹配字符串 he 和 The。

“[T]he” => The car is parked in the garage.
“[T]?he” => The car is parked in the garage.

Braces { }

在正则表达式中 {}是一个量词，常用来限定一个或一组字符可以重复出现的次数。 例如，表达式 [0-9]{2,3}匹配最少 2 位最多 3 位 0~9 的数字。

“[0-9]{2,3}” => The number was 9.9997 but we rounded it off to 10.0.

我们可以省略第二个参数。 例如，[0-9]{2,} 匹配至少两位 0~9 的数字。

“[0-9]{2,}” => The number was 9.9997 but we rounded it off to 10.0.

如果逗号也省略掉则表示重复固定的次数。 例如，[0-9]{3} 匹配3位数字。

“[0-9]{3}” => The number was 9.9997 but we rounded it off to 10.0.

Capturing Groups 特征标群

(...)里面的内容可以看作一个整体。例如，表达式 (ab)*匹配连续出现 0 或更多个 ab。如果没有使用 (...) ，那么表达式 ab* 将匹配连续出现 0 或更多个 b 。
我们还可以在 ()中用或字符 | 表示或。例如，(c|g|p)ar 匹配 car 或 gar 或 par.

“(c|g|p)ar” => The car is parked in the garage.

Non-Capturing Groups非特征标群

捕获组与捕获组不匹配，但捕获组与捕获组不匹配。非捕获组用一个?后跟：在括号(...)内。例如正则表达式(?:c|g|p)ar与(c|g|p)ar相似，因为它匹配相同的字符，但不会创建捕获组。

“(?:c|g|p)ar” => The car is parked in the garage.

Alternation 或运算符

或运算符就表示或，用作判断条件。
例如 (T|t)he|car匹配 (T|t)he 或 car。

“(T|t)he|car” => The car is parked in the garage.

Escaping Special Characters 转码特殊字符

反斜线\在表达式中用于转码紧跟其后的字符。用于指定 { } [ ] / \ + * . $ ^ | ?这些特殊字符。如果想要匹配这些特殊字符则要在其前面加上反斜线\。
例如 .是用来匹配除换行符外的所有字符的。如果想要匹配句子中的. 则要写成\.以下这个例子 \.?是选择性匹配.

“(f|c|m)at.?” => The fat cat sat on the mat.

Anchors 锚点

在正则表达式中，想要匹配指定开头或结尾的字符串就要使用到锚点。^指定开头，$指定结尾

The Caret ^

插入字符^用来检查匹配字符是否是输入字符串的第一个字符。

“(T|t)he” => The car is parked in the garage.

“^(T|t)he” => The car is parked in the garage.

The Dollar Sign $

美元字符$用来检查匹配字符是否是输入字符串的最后一个字符。

“(at.)” => The fat cat. sat. on the mat.

“(at.)$” => The fat cat. sat. on the mat.

Shorthand Character Sets 简写字符集

对于常用的字符集/正则表达式，有许多方便的速记，下表所示：

符号 描述除换行符外的所有字符
. Any character except new line
\w Matches alphanumeric characters: [a-zA-Z0-9_]匹配所有字母数字，等同于[a-zA-Z0-9_]
\W Matches non-alphanumeric characters: [^\w]匹配所有非字母数字，即符号，等同于:[^\w]
\d Matches digits: [0-9]匹配数字:[0-9]
\D Matches non-digits: [^\d]匹配非数字:[^\d]

\S Matches non-whitespace characters: [^\s]匹配所有非空格字符:[^\s]
\f Matches a page break匹配一个换页符
\n Matches a newline character匹配一个换行符
\r Matches a carriage return匹配一个回车符
\t Matches a tab匹配一个制表符
\v Matches a vertical tab匹配一个垂直制表符

Common RegExps

整数或者小数：^[0-9]+\.{0,1}[0-9]{0,2}$ 。

只能输入数字："^[0-9]*$"。

只能输入n位的数字："^\d{n}$"。

只能输入至少n位的数字："^\d{n,}$"。

只能输入m~n位的数字："^\d{m,n}$"

只能输入零和非零开头的数字："^(0|[1-9][0-9]*)$"。

只能输入有两位小数的正实数："^[0-9]+(.[0-9]{2})?$"

只能输入有1~3位小数的正实数："^[0-9]+(.[0-9]{1,3})?$"。

只能输入非零的正整数："^\+?[1-9][0-9]*$"。

只能输入非零的负整数："^\-[1-9][]0-9"*$。

只能输入长度为3的字符："^.{3}$"。

只能输入由26个英文字母组成的字符串："^[A-Za-z]+$"。

只能输入由26个大写英文字母组成的字符串："^[A-Z]+$"。

只能输入由26个小写英文字母组成的字符串："^[a-z]+$"。

只能输入由数字和26个英文字母组成的字符串："^[A-Za-z0-9]+$"。

只能输入由数字、26个英文字母或者下划线组成的字符串："^\w+$"。

验证用户密码："^[a-zA-Z]\w{5,17}$"正确格式为：以字母开头，长度在6~18之间，只能包含字符、数字和下划线。

验证是否含有^%&’,;=?$"等字符："[^%&',;=?$\x22]+"

只能输入汉字："^[\u4e00-\u9fa5]{0,}$"

验证Email地址："^\w+([-+.]\w+)*@\w+([-.]\w+)*\.\w+([-.]\w+)*$"。

验证InternetURL："^http://([\w-]+\.)+[\w-]+(/[\w-./?%&=]*)?$"。

验证电话号码："^(\(\d{3,4}-)|\d{3.4}-)?\d{7,8}$"正确格式为：”XXX-XXXXXXX”、”XXXX-XXXXXXXX”、”XXX-XXXXXXX”、”XXX-XXXXXXXX”、”XXXXXXX”和”XXXXXXXX”。

验证身份证号（15位或18位数字）："^\d{15}|\d{18}$"。

验证一年的12个月："^(0?[1-9]|1[0-2])$"正确格式为：”01”～”09”和”1”～”12”。

验证一个月的31天："^((0?[1-9])|((1|2)[0-9])|30|31)$"正确格式为；”01”～”09”和”1”～”31”。

匹配中文字符的正则表达式：[\u4e00-\u9fa5]
