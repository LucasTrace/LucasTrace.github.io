4.@Resource

@Autowired和@Resource的异同：

  • 都是用来自动装配的，都是可以放在属性字段上的
  • @Autowired和@Resource都可以通过ByName或者ByType查找，如果都匹配不到就会报错
  • @Autowired不能唯一匹配则需要通过@Qualifier(value=”XXX”)
  • @Resource不能唯一匹配则需要通过@Resource(name=”XXX”)
  • ？存在争议：默认是先用ByName

10.注解开发

  1. @Autowired 自动装配，放在属性字段上
  2. @Resource 自动装配，放在属性字段上
  3. @Nullable 字段标记这个注释，表示字段可以为null
  4. @Component 组件，放在类上，表示这个类被Spring扫描管理了，就是一个bean，id为类名的小写格式。等价于在配置文件中写<bean id="user" class="pojo.User"/> 注意@Component有三个衍生注解，我们在web开发中，会按照mvc三层架构分层：dao对应【@Repository】service对应【@Service】controller对应【@Controller】这三个个注解和@Component功能类似，都是代表将某个类注册到Spring中，进行装配
  5. @Value 赋值，等价于在配置文件中写<property name="name" value="pj"/>
  6. @Scope 确定作用域，放在类上

比较使用注解开发和用xml配置文件开发

  • 注解开发更简单，但是不是自己的类无法使用，且维护复杂
  • xml更加强大，适用于任何场合，但是代码量更大
  • 推荐使用xml来管理bean，使用注解完成属性的注入

12.面向切面编程（Aspect Oriented Programming）

SpringAOP的底层是代理模式，分为静态代理和动态代理

静态代理模式的优点：

  • 可以使真实的角色的操作更加纯粹，不用去关注一些公共的业务
  • 公共的就交给代理角色。实现业务的分工
  • 公共业务发生扩展时，方便集中管理

缺点：一个真实的角色产生时就需要产生一个代理角色，代码量会翻倍，开发效率会变低

动态代理分为：基于接口的动态代理：JDK动态代理，基于类的动态代理：cglib，以及java字节码实现：javassist

15.事务

  • 把一组业务当成一个业务来做，要么都成功，要么都失败
  • 事务在项目中非常重要，涉及到数据的一致性问题，不能马虎

事务ACID原则：

  • 原子性（Atomicity）
  • 一致性（Consistency）
  • 隔离性（Isolation）
  • 持久性（Durability）

一个使用 MyBatis-Spring 的其中一个主要原因是它允许 MyBatis 参与到 Spring 的事务管理中。而不是给 MyBatis 创建一个新的专用事务管理器，MyBatis-Spring 借助了 Spring 中的 DataSourceTransactionManager 来实现事务管理
